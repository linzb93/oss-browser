# 登录 / 首页视图测试用例

## 1. 账号列表显示

- [ ] **空状态**: 验证当账号列表为空时，是否显示“添加您的第一个账号”引导。
- [ ] **列表数据**: 验证账号列表是否正确显示 ID、名称和平台（例如“阿里云”）。
- [ ] **平台名称**: 验证平台类型 `1` 是否正确渲染为“阿里云”。

## 2. 添加账号

- [ ] **打开弹窗**: 点击“添加账号”按钮，验证弹窗打开且字段为空（默认平台除外）。
- [ ] **表单验证**:
    - [ ] 点击“提交”但不填写字段，验证所有必填字段（名称、Region、AK、SK、Bucket、Domain）的错误提示。
    - [ ] 输入无效的 Domain（非 http/https 开头），验证正则验证错误。
- [ ] **获取 Buckets**:
    - [ ] 在未填写 Region/AK/SK 时点击“获取”按钮，验证校验阻止请求。
    - [ ] 填写 Region/AK/SK 后点击“获取”，验证调用 `get-buckets` API。
    - [ ] 验证成功时 Bucket 下拉框已填充。
    - [ ] 验证失败时显示错误消息“信息输入错误，请重新输入”。
- [ ] **提交成功**: 填写所有有效信息，点击“提交”，验证调用 `login-save` API，显示成功消息，弹窗关闭，且列表刷新。

## 3. 编辑账号

- [ ] **打开编辑**: 点击某项的“编辑”，验证弹窗打开且表单已填充现有数据。
- [ ] **修改并保存**: 修改部分字段，点击“提交”，验证更新已保存并反映在列表中。

## 4. 账号操作

- [ ] **进入账号**: 点击“进入”，验证调用 `setDefaultAppId` 且页面重定向到 `/fileList` 并带有正确的 ID 参数。
- [ ] **复制账号**: 点击“更多” -> “复制”，验证弹窗打开且数据已填充但 ID 已清除（基于现有条目创建新条目）。
- [ ] **删除账号**:
    - [ ] 点击“更多” -> “删除”，验证出现确认弹窗。
    - [ ] 点击“取消”，验证未执行任何操作。
    - [ ] 点击“删除”（确认），验证调用 `home-removeAccount`，显示成功消息，且该项从列表中移除。

## 5. 边界情况与错误处理

- [ ] **API 失败**: 验证当 `home-getList`、`login-save` 或 `get-buckets` 失败时的优雅处理（错误提示）。
